The TSTP Format

Design Constraints

  • Compatible problem and solution language
    • Communicating systems
    • Pipelined filters
    • Common analysis tools' front-end
  • Sufficient information for solution presentation
  • Sufficient information for verification
    • Structural
    • Syntactic
    • Semantic
  • Easy to parse
    • Trivial in Prolog

Samples

  • Problem formula 
    fof(symmetry,axiom,
    ( ! [X,Y] :
        ( equal(X,Y)
       => equal(Y,X) ) ),
    theory(equality),
    [description('Symmetry is an axiom of equality')]).
  • Solution formula 
    cnf(32,derived,
    ( ~equal(compose(a,compose(b,A)),B)
    | ~equal(compose(a,compose(b,C)),B)
    | equal(A,C) ),
    inference(demod,[thm],[
        inference(back_demod,[thm],[13]),20,20,theory(equality)]),
    [iquote('back_demod,13,demod,20,20')]).

Generically

  • Flexible formula format 
    <language>(<name>,<type>,

    <optional source>,
